Set up your repository

After creating your repository in BitBucket or Stash etc. follow the directions they give you


Get latest code

git fetch

This gets all repositories form remote to your machine.

Does not merge into your repositories


Update your master repository

git merge origin master

Merge changes from 2 into master

Git Fetch and then Git Merge = Git pull

BUT pull always assumes master. If you are in a different branch this can be dangerous as it will try to merge into master still.


Create a feature branch

git checkout –B myFeatureBranch

Create a new branch from the head of the current branch (in this case master)

you can create this from any commit .... ever


Do work and commits

My preference is to use Git Extensions to commit as it handles all the adds and is more visual for partial commits.

Otherwise use

git add --all (or some matching expression)

it commit -m "commit message"


Keep your branch up-to-date with master (if you want to)

git rebase master

Rebase/Merge from master (after completing 2 + 3 again) (can use git merge master)

Merge and Rebase do the same thing i.e. merge all commits from the specified branch into the current one except,

Rebase groups all your commits together and shuffles them to the the top

Rebase can also "squash" all commits into one by using rebase -i (interactive rebase)

Could be any branch, not just master


Store your branch on remote (optional)

git push origin myFeatureBranch

If you want to store your branch at remote (long running feature branches for sharing or backing up)


When ready to merge back into master

git checkout master

git merge myFeatureBranch

git branch -D myFeatureBranch -delete branch (optional)

git push origin :myFeatureBranch -delete remote branch (optional)


Push changes to remote master

Git push origin master
